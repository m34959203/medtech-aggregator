## [Unreleased]

### Added (2026-06-27…28)
- **Витрина пациента**: краткое описание услуги (1586/1588 через Gemini) в карточке и на странице услуги; расстояние-«чекпоинт» (геопозиция в localStorage → дистанция до ближайшей клиники, сортировка «ближе» на главной/в выдаче); **сравнение одной услуги в 2–4 клиниках** на странице услуги (чекбоксы → панель → таблица цена/рейтинг/расстояние/онлайн-запись/адрес/режим/обновлено, лучшее подсвечено).
- **Дизайн «МедЦена»** (Claude Design import): зелёная гамма #0F8A7E, шрифт Golos Text, цветные категории-бейджи, стат-полоса на услуге, адаптив десктоп/мобайл.
- **Достоверность данных**: `Clinic.is_public` — обезличенные архив-клиники Кейса 2 скрыты из публичной выдачи (видны в MedArchive/админке); удалены услуги-сироты без цен; реальные адреса дозаполнены.
- **Админка оператора**: панель завершения приёма (статус + судьба позиций + CTA «Проверить N»/«Открыть прогон»/«Переобработать»/«Откатить»); управление источниками автосбора `GET/POST/PATCH/DELETE /api/ingest/sources` (`/admin/sources`); деталь прогона `GET /api/ingest/runs/{id}` (`/admin/runs/{id}`); **откат прогона** `POST /api/ingest/runs/{id}/rollback`; фильтр ревью по прогону `?run=` и по аномалиям `?filter=anomaly`; пикер клиник вместо UUID; авто-обновление дашборда; блок статуса WhatsApp; MedArchive — режим по умолчанию.
- **Ценовые аномалии**: детерминированный флаг `Price.is_anomaly` (нерезидент<резидент) — стабилен, в отличие от прежнего version-diff.
- **WhatsApp**: отправка координат клиники через наш Baileys-шлюз на ПК (`POST /api/wa/share-location`, флаг `transactional` обходит client-initiated, дневной лимит сохранён); на мобиле — нативный `wa.me`.

### Fixed (2026-06-27…28)
- **Очередь ревью прятала нераспознанные** позиции (`service_id IS NULL`) из-за INNER JOIN → счётчик «на проверке» рос, очередь была почти пустой; OUTER JOIN + обработка null в UI.
- **Плановый сбор** (`/api/ingest/run-scheduled`) больше не отдаёт 500 по таймауту прокси — запускается в фоне (BackgroundTasks), мгновенный ответ.
- **Счётчик городов** исключает пустой город `''` анонимных клиник (было 15 → стало 14).
- **Дубли-каноны** «Глюкоза (кровь)»/«…экспресс» слиты в SPECS-канон «Глюкоза (в крови)»; анти-рецидив в загрузчике справочника.

### Added
- **Семантическая нормализация (pgvector)**: эмбеддинги услуг (fastembed, multilingual), поиск по смыслу («кровь на сахар»→«Глюкоза»); на Postgres — pgvector `<=>`, иначе in-process; тир в нормализаторе + `/api/semantic/*`; грациозная деградация.
- **Postgres как прод-рантайм**: `docker-compose.prod.yml` поднимает `medtech-db`; движок с `pool_pre_ping` и скромным пулом; скрипт переноса данных `copy_to_pg.py` (SQLite→PG).
- **Rate-limit**: публичные POST (жалоба/лид/чат/корзина) и `auth/login` ограничены по IP (in-memory, `429` + `Retry-After`).
- **Авторизация админ-зоны** (passwordless): `ADMIN_TOKEN` + httpOnly-cookie; закрыты `/admin`, ревью, экспорт, приём данных, выдача порталов; вход по magic-link `?key=`. Fail-closed при незаданном токене.
- **Alembic-миграции**: схема управляется миграциями (`python -m app.migrate`), а не `create_all`; устойчиво к свежей/мигрированной/легаси БД; Postgres и SQLite.
- **Кейс 1 — пакетный приём архива** (`POST /api/ingest/upload-batch`): .zip или несколько прайсов за раз, клиника по префиксу имени `«<id>_файл»` или общему `clinic_id`, единый отчёт по всем файлам.
- **Экспорт каталога** (`GET /api/export/catalog?format=xlsx|csv`) — единый нормализованный свод как выходной артефакт обработки.
- **Админ-дашборд приёма** (`/admin`): статистика, экспорт, пакетная загрузка, журнал `ingestion_runs` (`GET /api/ingest/stats`, `/api/ingest/runs`).
- Связь карточек клиник с картой: клик по карточке центрирует карту и открывает балун клиники, клик по метке подсвечивает карточку в списке.
- Live-демо нормализации `/normalizer` (`POST /api/ingest/preview`) — сухой прогон fuzzy+LLM без записи в БД (антихардкод для жюри).

### Fixed
- Нормализатор не сливает «Глюкоза в моче»/«Креатинин в моче»/«Глюкозотолерантный тест» с кровяным аналитом — отдельные услуги (раньше моча искажала сравнение и «🏆 Лучшая цена»).
- Уверенность сопоставления на витрине теперь реальная (`token_set_ratio` к эталону, пол 0.82) вместо зашитых 100% у всех.
- Нормализатор не сливает повторный/онлайн/детский приём с первичным «Приём X» (отдельная услуга вместо ложного 100%-совпадения).
- Чат-помощник: распознавание города по падежам, фокус по явно названной услуге, ровно одна «🏆 Лучшая цена».
- Реальные адреса лабораторных сетей вместо заглушки «сеть лабораторий»; убран висячий разделитель в строке локации карточки.

## [0.1.0] — 2026-06

Первый сквозной MVP для Medtech Hackathon (Terricon Valley). Один продукт закрывает оба кейса.

### Added
- **Кейс 1 — приём прайсов.** Парсер xlsx/csv/pdf, веб-скрапер и REST-коннектор (два канала: push-загрузка и pull-автосбор), планировщик автосбора.
- **Нормализация к справочнику** — fuzzy-match (rapidfuzz) + опциональный LLM, дедупликация каналов с приоритетом официальной загрузки клиники.
- **Кейс 2 — агрегатор.** Витрина пациента на Next.js: поиск, сравнение цен по услуге между клиниками, фильтры по городу/категории/цене.
- **Карта клиник** (Яндекс.Карты) с метками цен и выделением лучшей цены.
- **Чат-помощник пациента** (`POST /api/chat`) на retrieval-injection поверх агрегатора; провайдер AlemLLM/Groq через `LLM_PROVIDER`, деградация в детерминированный поиск без ключа.
- Демо-данные через реальный конвейер (`app.seed`), набор демо-прайсов в `sample_data/`.
- `docs/`: архитектура, API, питч, правовой раздел об этике автосбора.
- CI (GitHub Actions): бэкенд-тесты (pytest) + сборка фронта.
