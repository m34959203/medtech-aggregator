# Участие в разработке

Спасибо за интерес к проекту! Ниже — короткие правила, чтобы изменения вливались быстро и без сюрпризов.

## Окружение

- **Backend:** Python 3.12, FastAPI, SQLAlchemy. По умолчанию SQLite — запускается без Docker.
- **Frontend:** Node 20, Next.js (App Router, TypeScript, Tailwind).
- Полный запуск — в разделе [«Быстрый старт» README](README.md#быстрый-старт).

```bash
# backend
cd backend && pip install -r requirements.txt && cp .env.example .env && python -m app.seed
# frontend
cd frontend && npm install && cp .env.example .env.local
```

## Ветки и коммиты

- Не коммитим напрямую в `main` — ветка под фичу/фикс: `feat/<кратко>`, `fix/<кратко>`.
- Коммиты — [Conventional Commits](https://www.conventionalcommits.org/), **префикс на английском, описание на русском**, глагол в инфинитиве, без точки в конце, заголовок ≤ 72 символа:

```
feat(compare): связать карточки клиник с картой
fix(normalizer): не писать в БД при сухом прогоне preview
docs: обновить README — раздел Quick Start
test(ingestion): добавить кейс на дедупликацию каналов
```

## Перед PR

- [ ] Бэкенд-тесты зелёные: `cd backend && python -m pytest -q`
- [ ] Фронт собирается: `cd frontend && npm run build`
- [ ] Линтер фронта чист: `cd frontend && npm run lint`
- [ ] `.env.example` актуален, если добавлены новые переменные
- [ ] В коммит не попали реальные секреты (ключи, токены, дампы БД)
- [ ] Документация (README / `docs/`) обновлена, если поведение изменилось

PR оформляем по шаблону (`Что сделано / Зачем / Как проверить / Чеклист`) — он подставится автоматически.

## Issues

Используйте шаблоны `bug` / `feature` — они появятся при создании issue. Чем точнее шаги воспроизведения и окружение, тем быстрее фикс.

## Безопасность

Никогда не коммитьте реальные `.env`, API-ключи и дампы. Если секрет всё же утёк — **ротируйте ключ**, удаления коммита недостаточно.
