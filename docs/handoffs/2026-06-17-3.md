# Handoff 2026-06-17 #3 — live-демо нормализации (антихардкод)

## Контекст
Критика жюри: «точность 100% выглядит захардкоженной», т.к. bulk-сид идёт через
детерминированную карту `SPECS` (conf=1.0), а умный движок (fuzzy+LLM) не виден.

## Что сделал (Claude) — задеплоено на прод
- **Страница `/normalizer`** (`frontend/app/normalizer/page.tsx`) — жюри вводит любые кривые
  названия услуг → видит вживую, как движок сводит их к справочнику: эталон, **уверенность**
  (полоса) и **метод** (нечёткое сопоставление / LLM / новая услуга). Вход контролирует
  пользователь → хардкод исключён.
- **`POST /api/ingest/preview`** (`backend/app/routers/ingestion.py`, `Normalizer.preview()`) —
  сухой прогон того же движка **без записи в БД**.
- **`backend/app/llm.py`** — общий провайдер-агностичный LLM-доступ (AlemLLM/Groq); LLM-шаг
  нормализатора теперь работает и на AlemLLM (в проде ключ Alem).
- **`backend/sample_data/dirty_pricelist.csv`** — «грязный» прайс для live-upload на сцене.
- Тест preview (non-mutating) → **24 pytest**. Прод проверен: `/normalizer` 200, preview отдаёт
  корректные результаты.

## Сценарий защиты (рекомендация)
1. Открыть **`https://medtech.technokod.kz/normalizer`** — там уже предзаполнены кривые названия
   (ОАК с лейкоформулой, «сахар крови», «УЗ-исследование почек» и т.д.). Нажать «Нормализовать».
   Показать: одно и то же сводится к эталону, разные методы и уверенность, «УЗ-исследование почек»
   разрешает **LLM**.
2. Дать жюри **самим набрать** любое название — движок отработает на лету.
3. (Опц.) Live-upload `dirty_pricelist.csv` через `/api/ingest/upload` → услуги появляются в витрине.
4. Нарратив: двухуровневость намеренная — курируемый сид для качества демо + боевой fuzzy+LLM-движок
   для приёма любых новых прайсов (он же на `/normalizer`).

> ⚠️ Гоча движка: fuzzy не разворачивает аббревиатуры (например «магнитно-резонансная томография»
> нечётко сводится к «КТ», а не «МРТ») — такие примеры в предзаполнение НЕ клал. Если жюри введёт
> подобное — это честно: именно поэтому есть LLM-уровень; можно прокомментировать.
