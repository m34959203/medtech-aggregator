## Кейс 1 — MedPrice (агрегатор, цель 100%)

### §3.1 Парсер
- [ ] Ручной автосбор: /admin → «Автосбор с сайта» → URL клиники (напр. invitro.kz) → запустить → в «Журнале прогонов» новая строка `status=normalized`, items>0.
- [ ] Дедуп: повторный сбор того же источника НЕ плодит дубль-цены (кол-во цен не растёт кратно).
- [ ] Журнал ошибок: скорми битый URL → строка прогона `status=error` с причиной.
- [ ] Raw-слой: у прогона есть `raw_content` (сырой текст) — видно в деталях прогона.
- [ ] Cron: `crontab -l` на сервере содержит `cron-ingest.sh` каждые N часов.

### §3.2 Нормализация
- [ ] Поиск «ОАК», «CBC», «Общий анализ крови» → одна и та же услуга.
- [ ] Очередь unmatched: /admin/review показывает несопоставленные позиции, confirm/reassign/reject работают.

### §3.3 UI поиск/сравнение (главная)
- [ ] Поиск с автодополнением по справочнику.
- [ ] Фильтры: город, категория, ценовой диапазон (min/max), рейтинг, онлайн-запись — каждый меняет выдачу.
- [ ] Результат: цена, адрес, режим работы, ссылка на источник.
- [ ] Сортировки: цена ↑/↓, расстояние (с геолокацией), дата обновления.
- [ ] Карточка клиники: все услуги, контакты, сайт, дата обновления цены.
- [ ] Мобильная вёрстка (узкий экран) — не ломается.

### §3.4 Доп-функции
- [ ] Карта клиник с маркерами.
- [ ] Подписка на снижение цены (форма «🔔 Подписаться»).
- [ ] Сравнение таблицей: /compare — 2–4 клиники × услуги, 🏆 лучшая цена.
- [ ] История цены: услуга с ≥2 датами → график динамики.
- [ ] Маршрут до клиники (кнопка «Маршрут» → Яндекс.Карты).

### §4 Нефункциональные
- [ ] Поисковая выдача < 3 c.
- [ ] Цена старше 30 дней не помечена как актуальная.
- [ ] Падение одного источника не валит остальные (отказоустойчивость).

## Кейс 2 — MedArchive (обработка архива, для полноты)

### §2.1/§4.1 Приём архива
- [ ] /admin → карточка приёма → включить **чекбокс «Режим MedArchive (Кейс 2)»**.
- [ ] Загрузить .zip с прайсами (взять `docs/Прайс/` или собрать zip из 4 форматов) с общим clinic_id → один отчёт по файлам.
- [ ] Авто-определение типа файла (формат показан у каждого файла в отчёте).
- [ ] **Оригиналы сохранены**: у файлов значок 💾, в ответе «оригиналов сохранено: N»; на сервере `docker exec medtech-backend ls /data/uploads/<run_id>/` — файл на месте.

### §4.2 Извлечение по форматам
- [ ] PDF-текст: таблицы распознаны.
- [ ] PDF-скан: OCR отработал (позиции извлечены).
- [ ] XLSX: все листы, шапка не в первой строке найдена.
- [ ] DOCX + tracked changes: правки приняты, финальный текст.

### §4.3 Нормализация
- [ ] Сопоставление точное+синонимы+нечёткое; непривязанные → /admin/review.

### §4.4 Валидации + версионирование
- [ ] Резидент/нерезидент раздельно (в карточке/партнёрском API price_resident≠price_nonresident).
- [ ] Аномалия >50%: повторная загрузка с ценой ×2 → в отчёте «аномалий: N».
- [ ] Версионирование: история цены пополняется, старая не удаляется.

### §4.1 Повторная обработка
- [ ] `POST /api/ingest/archive/{run_id}/reprocess` (из сохранённого оригинала) → 200, items>0.

### §4.5 API (Swagger /docs)
- [ ] `GET /api/services`, `/api/services/{id}/partners`, `/api/partners`, `/api/partners/{id}/services`, `/api/search`, `/api/unmatched`, `POST /api/match` — все отвечают.
