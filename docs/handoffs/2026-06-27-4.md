### [BUG] критично — очередь ревью прятала нераспознанные
- Очередь делала **INNER JOIN** на справочник → позиции с `service_id IS NULL`
  (нераспознанные) в неё НЕ попадали. На проде: счётчик «на проверке» = 6132, а
  в очереди реально было **~5** (с услугой), **6127 нераспознанных — без пути к разбору**.
- Фикс: **OUTER JOIN** + обработка null в UI («не распознано — назначьте услугу»,
  «Подтвердить» выключена пока не назначена услуга). Теперь очередь = 6132,
  фильтр по прогону отдаёт реальные N (напр. прогон #1276 → 621). Счётчик и очередь
  согласованы.
