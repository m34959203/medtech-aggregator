# Правовой и этический аспект автосбора (② pull)

Веб-парсинг чужих сайтов — зона рисков. Жюри это оценит, если показать осознанность.

## Принципы, заложенные в платформу
- **Вежливый сбор.** `web_scraper` ходит с честным `User-Agent`, таймаутом и без параллельной бомбардировки; уважаем `robots.txt` и пользовательские соглашения, не создаём чрезмерную нагрузку.
- **Приоритет официальных каналов.** Целевая модель — официальная загрузка (① push) и API клиник-партнёров. Парсинг используется как временный **bootstrap**, пока клиника не подключилась напрямую. Дедупликация автоматически отдаёт приоритет официальной загрузке над спарсенными данными.
- **Прозрачность происхождения.** У каждой цены хранится `source_type` (upload / web_scrape / api) и `valid_from`. В интерфейсе источник помечается: «Официально от клиники» / «С сайта клиники от такого-то числа» / «API».
- **Не выдаём спарсенное за официальное.** На витрине справочные данные с сайтов явно маркированы и сопровождаются датой сбора.
- **Цены медуслуг — публичная информация**, но безопаснее приоритизировать партнёрские договорённости.

## Фраза для питча
> «Автосбор — для быстрого наполнения базы; целевая модель — официальная загрузка и API клиник-партнёров. Источник и дата каждой цены видны пациенту.»
