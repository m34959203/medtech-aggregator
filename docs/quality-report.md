## Методика

1. **Извлечение** — DOCX (приём tracked changes: `<w:del>` удаляются, `<w:ins>`
   раскрываются), XLSX/XLS (все листы, автодетект строки-заголовка, многострочная
   шапка), PDF (таблицы → реконструкция по координатам слов), PDF-скан → OCR
   (tesseract rus/kaz/eng). Резидент/нерезидент извлекаются раздельно.
2. **Нормализация** — code-first по коду тарификатора (точное совпадение → conf=1.0),
   иначе нечётко (rapidfuzz token_set) → семантика (offline-эмбеддинги, pgvector) →
   LLM-арбитраж (Gemini 2.5-flash / Vertex). Биоматериал-гард: стул/мокрота/мочевой
   вариант не сливаются с кровяной услугой.
3. **Валидации (§4.4)** — цена > 0 и ≤ 100 млн KZT (отсечка OCR-артефактов),
   нерезидент ≥ резидент (флаг), аномалия > 50% к прошлой версии, имя не пустое;
   версионирование цен (история не удаляется).
4. **Очередь ревью** — позиции ниже порога уверенности → оператору
   (`GET /api/unmatched`, экран `/admin/review`).
5. **Сохранение оригиналов (§2.1/§5)** — исходные файлы складываются в
   `/data/uploads/<run_id>/` (постоянный том) для повторной обработки и аудита;
   `POST /api/ingest/archive/{run_id}/reprocess` перезапускает разбор из оригинала.

## Остаток (честно)

- Хвост загрязнения синонимов (наследие обратной связи нормализатора) частично
  вычищен (сняты услуги-магниты); единичные trace-element/antibody-кейсы лечатся
  LLM-verified recheck-проходом (`POST /api/review/recheck`).
- Документы с низким auto-rate (Клиника 2 2025, Клиника 7) — операторская очередь
  ревью работает штатно; остаток нормализуется дозаведением кодов/синонимов.
