# Roadmap

Куда движется проект после хакатонного MVP. Статусы поддерживаются вручную.

## ✅ Сделано — MVP (хакатон Terricon, июнь 2026)

Один сквозной продукт закрывает оба кейса:

- **Приём прайсов (Кейс 1):** парсер xlsx/csv/pdf, веб-скрапер, REST-коннектор; два канала (push-загрузка клиникой и pull-автосбор по расписанию).
- **Пакетная обработка архива:** `upload-batch` принимает .zip / несколько прайсов → единый отчёт (прямой ответ на «обработку архива прайсов»).
- **Экспорт каталога:** единый нормализованный свод в xlsx/csv — выходной артефакт Кейса 1.
- **Админ-дашборд приёма (`/admin`):** статистика, пакетная загрузка, экспорт, журнал `ingestion_runs`.
- **Нормализация + дедупликация:** приведение разнобоя названий услуг к единому справочнику (fuzzy + опциональный LLM); при конфликте приоритет у официальной загрузки клиники; гард не-первичных приёмов.
- **Агрегатор (Кейс 2):** витрина пациента — поиск, сравнение цен между клиниками, фильтры, карта клиник с лучшей ценой; карточки связаны с картой.
- **Чат-помощник:** диалоговый поиск по витрине на retrieval-injection (бот не выдумывает цены); распознавание города по падежам.
- **Live-демо движка:** страница `/normalizer` — сухой прогон нормализации без записи в БД.

## 🔜 Ближайшее

- **OCR сканов** — приём прайсов фото/сканом (сейчас только текстовый слой PDF).
- **Личный кабинет клиники** — ручная модерация сопоставлений с уверенностью ниже порога `MATCH_CONFIDENCE_THRESHOLD`.
- **Устойчивый pull-сбор** — Playwright для JS-сайтов клиник, ретраи и расписание через Celery/cron, метрики свежести цен.
- **Качество нормализации** — расширение справочника услуг, синонимы, разбор составных прайс-позиций.

## 🌅 Дальше

- **Прод-релиз `v1.0.0`** — Postgres по умолчанию, наблюдаемость, rate-limit на публичные эндпоинты.
- **Гео и логистика** — фильтр «дешевле + ближе» с учётом маршрута, а не только координат.
- **Аналитика для клиник** — позиционирование цены относительно рынка по каждой услуге.

> Приоритеты могут меняться по обратной связи жюри и первых клиник. Предложения — через [issue с шаблоном feature](../.github/ISSUE_TEMPLATE/feature.md).
